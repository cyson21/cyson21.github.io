# 0001. Astro 정적 사이트와 증거 중심 콘텐츠 구조

- 상태: accepted
- 날짜: 2026-07-14

## 배경

기존 이력서와 포트폴리오 HTML은 PDF 출력을 위한 문서형 레이아웃입니다. 공개 웹사이트는 여러 화면 크기에서 빠르게 탐색하고, 프로젝트별 코드와 테스트 근거를 직접 확인할 수 있어야 합니다.

## 결정

- 공개 사이트는 기존 생성 흐름과 분리된 독립 저장소로 운영합니다.
- Astro 정적 출력과 Content Collections를 사용합니다.
- 프로젝트 페이지는 프로젝트 개요, 기술 선택, 주요 구현, 관련 테스트, 테스트 결과와 프로젝트 범위 순서를 공통으로 사용합니다.
- 동적인 기능은 흐름 모드와 코드 근거 탭처럼 이해에 필요한 범위로 제한합니다.

## 대안

- 기존 생성 HTML 재사용: 인쇄 레이아웃과 공개 웹 탐색 구조가 서로 달라 제외했습니다.
- Next.js 기반 단일 페이지: 서버 기능이 필요 없고 프로젝트별 구조화 콘텐츠가 중심이라 제외했습니다.

## 결과

기존 제출 문서는 그대로 유지하면서 공개 사이트의 접근성, 반응형 구성, 공개 안전 검사를 독립적으로 발전시킬 수 있습니다.
