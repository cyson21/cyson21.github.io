01

해결한 문제와 담당 역할

한정된 재고를 여러 요청이 동시에 주문할 때, 주문 생성과 재고 예약, 결제 승인 중 하나만 실패해도 서비스별 상태가 달라질 수 있습니다. 여기에 동일 이벤트의 재전달과 브로커 중단이 겹치는 조건을 실패 시나리오로 정의했습니다.

주문·재고·결제 서비스 경계, Saga·Outbox, 게이트웨이 인증과 장애 복구 흐름 직접 설계·구현을 담당했습니다. 각 서비스의 로컬 트랜잭션은 유지하되, 전체 흐름은 보상과 재처리로 수렴하도록 구성했습니다.

책임 경계 서비스 간 상태 전이를 직접 설계하고, 중복 처리 방지와 실패 상태 보존까지 테스트로 확인했습니다.
02

기술 선택과 이유

선택 적용 이유
Saga 상태 조율 주문 상태를 기준으로 재고 예약과 결제 결과를 전이 분산 트랜잭션 없이도 실패 지점과 보상 상태를 명시적으로 남기기 위해 선택했습니다.
트랜잭션 Outbox 도메인 변경과 발행할 이벤트를 같은 로컬 트랜잭션에 기록 DB 반영 후 Kafka 발행이 끊기는 이중 쓰기 구간을 줄이고 재전송 경로를 보존하기 위해 선택했습니다.
