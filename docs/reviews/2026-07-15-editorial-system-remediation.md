# 공개 포트폴리오 편집 체계 개선 기록

## 필수 반영 67건

1. 사이트 성격을 마케팅 랜딩페이지에서 기술 사례 기록으로 재정의했다.
2. 페이지 H1 최대 크기를 60px 이하로 제한했다.
3. 모바일 H1 최소 크기를 34px 이상으로 고정했다.
4. 본문 기본 크기를 18px로 올렸다.
5. 모바일 본문 크기를 17px로 올렸다.
6. 보이는 보조 텍스트의 최소 크기를 13px로 정했다.
7. 한글 제목의 단어 단위 줄바꿈을 유지했다.
8. 글자 간격을 0으로 통일했다.
9. 섹션 상하 여백을 공통 토큰으로 정리했다.
10. 섹션 제목을 구분선 기반 구조로 통일했다.
11. 번호가 있는 섹션 제목의 열 너비를 고정했다.
12. 모바일 섹션 제목을 한 열 흐름으로 전환했다.
13. 본문 폭을 780px로 제한해 긴 문장의 가독성을 높였다.
14. 보조색을 WCAG AA 대비를 넘도록 진하게 조정했다.
15. 헤더의 `SC` 상자 로고를 제거했다.
16. 이름과 직무를 조합한 워드마크를 만들었다.
17. 헤더 탐색을 프로젝트·경력·이력서로 한국어화했다.
18. 현재 페이지 탐색 표시를 하단선으로 단순화했다.
19. 모바일 메뉴 버튼을 44px 터치 영역으로 유지했다.
20. 공통 페이지 인트로를 좌측 라벨·우측 본문 구조로 바꿨다.
21. 페이지 인트로 제목 크기를 3.1rem로 낮췄다.
22. 페이지 라벨을 실제 페이지 목적을 밝히는 한국어로 바꿨다.
23. 인트로 설명 폭을 제한해 줄 길이를 안정화했다.
24. 홈 첫 화면 제목을 이름 `손찬양`으로 단순화했다.
25. 홈의 작은 키워드와 거대 슬로건 조합을 제거했다.
26. 홈 프로필 사진을 92px로 조정해 이름과 균형을 맞췄다.
27. 홈 요약 본문을 1.22rem로 올렸다.
28. 홈 우측 패널을 세 가지 상태 보호 기준으로 재작성했다.
29. 홈 우측 패널을 카드 대신 선형 기록 구조로 바꿨다.
30. 홈 섹션을 검증 결과·대표 프로젝트·문제 해결 방식·경력 순으로 번호화했다.
31. 검증 결과 제목을 추상 키워드 대신 결과 중심으로 바꿨다.
32. 대표 프로젝트 설명을 문제와 연결 테스트 중심으로 정리했다.
33. 여섯 프로젝트 문제 지도를 홈에 추가했다.
34. 문제 지도의 영문 단계를 요청·상태·근거·실패·복구·시점으로 한국어화했다.
35. 문제 지도의 개별 카드 그림자와 둥근 상자를 제거했다.
36. 문제 해결 방식을 네 개의 평평한 기록 셀로 재구성했다.
37. 경력 앞의 `운영 백엔드와 파이프라인` 추상 제목을 제거했다.
38. 홈 경력을 데이터 파이프라인에서 운영 백엔드로 이어지는 독립 섹션으로 바꿨다.
39. 프로젝트 행의 인덱스, 태그, 수정일과 링크 글씨를 키웠다.
40. 모바일 프로젝트 행에서 요약문을 숨기지 않도록 바꿨다.
41. `Case study only`를 `사례 문서`로 한국어화했다.
42. 검증 결과 카드의 라벨·결과·근거 경로를 키웠다.
43. 검증 결과 카드의 표현식을 제목보다 작게 조정했다.
44. 설계 판단 표의 인덱스·항목·본문을 키웠다.
45. 코드 근거의 파일 경로와 테스트 경로를 키웠다.
46. 코드 근거의 영문 대문자 변환을 제거했다.
47. 코드 발췌 크기를 0.84rem로 올렸다.
48. 흐름 탭과 단계 라벨을 키웠다.
49. 프로젝트 구성도 캡션과 노드 글씨를 키웠다.
50. 프로젝트 목록 제목을 `프로젝트`로 직접 표기했다.
51. 프로젝트 필터를 전체·백엔드·데이터·AI로 한국어화했다.
52. 프로젝트 필터의 결과 수와 상태 문구를 키웠다.
53. 검증 수준 안내를 독립된 하단 정보 밴드로 구성했다.
54. 경력 페이지 제목을 `경력`으로 직접 표기했다.
55. 경력 개요를 고정 사이드바에서 전체 폭 요약 행으로 바꿨다.
56. 회사별 경력을 문제·조치·확인 순서로 정리했다.
57. 경력 카드의 제목과 세부 항목 크기를 올렸다.
58. 업무 원칙을 재현·경계·검증의 번호 섹션으로 분리했다.
59. 웹 이력서 제목을 이름 `손찬양`으로 단순화했다.
60. 웹 이력서의 Contact·Profile·Experience·Skills·Education을 한국어화했다.
61. 웹 이력서 각 섹션에 같은 상단 구분선과 간격을 적용했다.
62. 프로젝트 상세 H1을 4.7rem에서 3.4rem로 낮췄다.
63. 프로젝트 상세 메타데이터·목차·검증표를 한국어화했다.
64. 프로젝트 상세의 문제·설계·규칙·근거·검증 라벨을 같은 문법으로 맞췄다.
65. 제외 범위와 후속 계획의 중복 제목을 분리했다.
66. 주요 5개 경로의 가로 넘침·접근성·한 개 H1을 자동 검사했다.
67. 보이는 텍스트 13px 이상과 H1 34~60px 범위를 E2E 회귀 검사로 추가했다.

## 선택 유지 또는 보류

- **외부 웹폰트**: 개인정보·성능·네트워크 의존성을 늘리므로 시스템 글꼴을 유지한다.
- **다크 모드**: 현재 제출·채용 문서의 일관성에 직접 기여하지 않아 보류한다.
- **스크롤 애니메이션**: 내용 탐색을 방해할 수 있어 적용하지 않는다.
- **숙련도 차트**: 객관적 척도가 없어 의미 없는 자기평가가 되므로 적용하지 않는다.
- **방문자 분석 도구**: 공개 전 개인정보 정책과 목적을 먼저 정해야 하므로 보류한다.
- **영문 전체 번역**: 유지 비용이 크므로 실제 영문 지원 수요가 생길 때 별도 콘텐츠로 만든다.
- **추가 사진·일러스트**: 실제 작업물보다 장식 비중을 키우므로 현재 프로필과 프로젝트 시각 자료만 사용한다.
